# Model-Hub: Локальный AI Ассистент Кода

Model-Hub - это разрабатываемый локальный AI ассистент кода, предназначенный для работы на Windows 11 без необходимости подключения к внешним серверам. Проект использует фреймворк `burn` для работы с моделями машинного обучения.

## Основные Возможности (Планируемые)

*   **Локальный Инференс LLM:** Вся обработка моделей происходит на машине пользователя.
*   **Графический Интерфейс Пользователя (GUI):** Встроенный редактор кода с интеллектуальными функциями.
*   **Поддержка LSP (Language Server Protocol):** Для предоставления функций автодополнения, диагностики, hover-информации и т.д.
*   **Интерфейс Командной Строки (CLI):** Для выполнения задач инференса из консоли.
*   **Фокус на Gemma:** Первоначальная реализация ориентирована на модели семейства Gemma.

## Технологический Стек

*   **Язык:** Rust
*   **ML Фреймворк:** `burn` (версия 0.17.0)
*   **GUI Фреймворк:** `egui` с `eframe`
*   **Асинхронность:** `tokio`
*   **LSP:** Реализация сервера и клиента для взаимодействия.

## Структура Проекта

Проект организован как Rust workspace со следующими основными крейтами:

*   `utils_crate`: Общие утилиты, структуры данных для задач и параметров.
*   `core_burn`: Ядро для определения архитектур моделей на `burn` (например, Gemma) и низкоуровневых компонентов (KV-кэш, RoPE).
*   `model_loader`: Загрузка весов моделей (SafeTensors, MPK) и их конфигураций.
*   `inference_engine`: Движок для выполнения инференса моделей.
*   `lsp_server_backend`: Реализация LSP сервера, который использует `inference_engine`.
*   `gui_app`: Графическое приложение, взаимодействующее с `lsp_server_backend`.
*   `cli_app`: Консольное приложение для инференса.
*   `(опционально) api_server_crate`: Локальный HTTP API сервер.

## Текущий Статус

*Детальное описание текущего статуса разработки каждого модуля.*
*   `utils_crate`: Готов.
*   `core_burn`: API и структура готовы. **Требуется реализация ML-логики `forward` методов для Gemma.**
*   `model_loader`: API и структура готовы. Загрузка SafeTensors требует завершения маппинга на `Record`.
*   `inference_engine`: API и структура готовы. Функциональность зависит от `core_burn`.
*   `lsp_server_backend`: В процессе разработки.
*   `gui_app`: Основа UI и взаимодействия с LSP заложена.
*   `cli_app`: На начальном этапе.

## Установка и Сборка (Предварительно)

1.  Установите Rust: [https://www.rust-lang.org/tools/install](https://www.rust-lang.org/tools/install)
2.  Убедитесь, что у вас настроена версия Rust, указанная в `rust-toolchain.toml`.
3.  Клонируйте репозиторий (если бы он был): `git clone ...` (для локального проекта этот шаг не нужен, если вы уже работаете в директории).
4.  Для сборки всего воркспейса:
    ```bash
    cargo build --workspace --release
    ```
5.  Для запуска GUI (пример, после сборки):
    ```bash
    cargo run --package gui_app --release
    ```

## Загрузка Моделей

Модели должны быть загружены локально. Пути к моделям и конфигурациям указываются при запуске `lsp_server_backend` или `cli_app`, или через настройки в `gui_app`.

Пример структуры директории модели (`.model_cache/gemma-7b-it/`):
*   `config.json`
*   `model.safetensors.index.json` (если модель шарднута)
*   `model-00001-of-0000X.safetensors`
*   `tokenizer.json`
